# Product Proposal: Robot Learning Logger

## 1. Обоснование идеи

Современные роботы обучаются поведению через RL/IL, но качество данных — критический фактор.  
Ошибки сенсоров, дублирование логов или повреждённые фреймы делают процесс обучения нестабильным.  
Проект направлен на создание агента, способного контролировать этот процесс и выдавать раннюю обратную связь о качестве данных.

## 2. Цель и метрики

**Цель:**  
Автоматизировать сбор и оценку качества обучающих данных робота.

**Метрики успеха:**

| Категория | Метрика | Цель |
|------------|----------|------|
| Продуктовая | Доля автоматически верифицированных логов | ≥ 80 % |
| Агентская | Среднее время анализа одного сета данных | ≤ 10 с |
| Техническая | Точность детекта "плохих" данных | ≥ 90 % по ручной валидации |

## 3. Потенциальные сценарии использования

| Сценарий | Описание | Edge‑кейсы |
|-----------|-----------|------------|
| `data_collection` | Агент получает данные из ROS‑топиков | Потеря сообщения / таймштамп рассинхронизирован |
| `quality_check` | Агент анализирует структуру и шумность данных | Сенсор замер на несколько секунд или перегрузился |
| `annotation` | Генерация комментария (“робот застрял на 03:12…”) | LLM произвёл галлюцинацию |
| `dataset_versioning` | Сохранение версии + отчёта | Повреждённые файлы или нехватка места |

## 4. Ограничения

**Технические SLO:**
- p95 latency проверки: ≤ 15 секунд на эпизод;  
- допустимая потеря данных: < 0.1%;  
- хранение истории (1 ГБ лимит в PoC).

**Операционные:**
- 2 человека × 2 недели;  
- бюджет на API LLM — ≤ 100 USD;  
- симуляция без реальных устройств.

## 5. Архитектурный набросок

**Компоненты системы:**
1. **ROS Interface** — подписчик на топики `/odom`, `/imu`, `/camera`.
2. **Data Collector** — буфер и сохранение данных.
3. **Quality Evaluator (AI Agent)** — анализ последовательностей и формирование отчётов.
4. **Memory Hub** — слои данных и метаданные (SQLite/JSON).
5. **Monitoring API** — REST‑точка, отдающая статистику.

**Интеграции:**
- `roslibpy` / `rclpy` для ROS;
- LLM API (OpenAI / Anthropic / Llama cpp);
- локальная БД / MinIO‑облако для хранения;
- Grafana/Plotly Dash для визуализации.

**Data flow:**

- LLM‑агент и эвристики анализируют данные, но не выполняют запись.  
- Все операции с файлами выполняет «технический» модуль (sandbox).

## 6. Возможности расширения
- Поддержка мульти‑роботов (сети агентов).  
- Встраивание в ML‑пайплайн: автоматическая подготовка обучающих батчей.  
- Интеграция с MLOps‑платформами (Weights & Biases, MLflow).
