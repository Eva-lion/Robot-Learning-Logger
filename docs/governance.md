# Governance & Risk Register — Robot Learning Logger

## 1. Risk Register

| Риск | Вероятность | Влияние | Детект | Защита | Остаточный риск |
|------|--------------|----------|---------|---------|-----------------|
| 1. Утечка изображений с людьми | Средняя | Высокое | Обнаружение лиц (OpenCV, hash‑check) | Замыливание / обрезка кадров | Низкий |
| 2. Потеря соединения с ROS | Средняя | Среднее | Таймаут ping | Retry + локальный буфер | Низкий |
| 3. Повреждение данных при записи | Низкая | Среднее | Проверка CRC / hash | Повторная запись | Низкий |
| 4. Ошибочная аннотация LLM | Высокая | Среднее | Вручную проверяем 10 % отчётов | Feedback + дообучение | Средний |
| 5. Перегрузка API / токен‑лимит | Средняя | Низкое | Метрика latency | Ограничение запросов и кэширование | Низкий |

## 2. Политика логирования

- Все логи сохраняются локально, без передачи в публичные облака.
- Идентификаторы сессий и пользователей хэшируются.
- В логах хранятся только агрегированные метаданные.

## 3. Работа с персональными данными

- В PoC используется только симуляция и обезличенные данные.
- Если появятся кадры с человеком — применяется авто‑анонимизация.

## 4. Защиты от инъекций и подтверждения действий

- Все LLM‑вызовы проходят через санитайзер → запрещены команды filesystem/exec.  
- LLM‑агент не имеет прямого доступа к ROS‑интерфейсу записи.
- Любые изменения состояния датасета требуют ручного подтверждения (человеческий approval).
